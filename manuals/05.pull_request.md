# Pull Requestを送る

Pull Requestを送り、CI/CDによるステージング環境を利用する。

## 1. フォーク元リポジトリをupstreamとして登録する

クローンしたリポジトリ（自分のFork）に対して、フォーク元のリポジトリを `upstream` として登録する。

```bash
git remote add upstream https://github.com/genai-docs/genai-mkdocs-hands-on.git
```

登録されたリモートを確認する。

```bash
git remote -v
```

## 2. 作業用ブランチを作成する

```bash
git switch -c feature/your-branch-name
```

## 3. 変更をコミットしてForkへプッシュする

```bash
git add .
git commit -m "コミットメッセージ"
git push -u origin feature/your-branch-name
```

## 4. フォーク元へのPull Requestを作成する

GitHub CLI (`gh`) を利用してフォーク元 (`genai-docs/genai-mkdocs-hands-on`) に対してPRを作成する。

```bash
gh pr create \
  --repo genai-docs/genai-mkdocs-hands-on \
  --base main \
  --head <your-account>:feature/your-branch-name \
  --title "PRタイトル" \
  --body "PRの説明"
```

`<your-account>` は自分のGitHubアカウント名に読み替える。

ブラウザから作成する場合は、GitHub上の **自分のFork** （`https://github.com/<your-account>/genai-mkdocs-hands-on`）のページを開く。

`Contribute` → `Open pull request` を選択する。遷移先で base が `genai-docs/genai-mkdocs-hands-on` の `main`、compare が自分のForkの作業ブランチになっていることを確認する。
